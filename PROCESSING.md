# PatientTables 数据处理说明

本文档说明 `main.py` 对原始数据所做的每一步处理，以及处理的原因和细节。

---

## 输出文件一览

| 文件 | 说明 |
|------|------|
| `2022_clean.csv` / `2023_clean.csv` / `2024_clean.csv` / `stroke_clean.csv` | 原始数据去除空行、空列后的版本，保留所有原始字段 |
| `2022_with_id.csv` / `2023_with_id.csv` / `2024_with_id.csv` / `stroke_with_id.csv` | 经过完整处理的分析版本，含 person_id，见下方处理步骤 |
| `patient_reference.csv` | 患者主索引：person_id → 姓名、性别、出生日期、出现来源 |

所有文件输出至 `/Users/haochen/Desktop/PatientTables/output/`，编码 UTF-8 with BOM（可直接用 Excel 打开）。

## 处理步骤详解

### Step 1 — 加载原始数据，去除空行

**对象：** 2022.csv、2023.csv、2024.csv、2022-2024.xlsx（脑卒中 sheet）

**操作：**
- CSV 文件以 GBK 编码分块读取（每块 50,000 行），去除所有字段均为空的行
- Excel 文件直接读取，同样去除全空行
- 脑卒中 xlsx 的性别列原始值为 `"1 男"` / `"2 女"`，提取其中的 `男`/`女` 字符

**原因：**
Excel 导出 CSV 时会将文件填充至最大行数（1,048,575 行）。三个年度 CSV 中约有 975,000 行是完全空白的填充行，需要去除。

**列名修复：**
部分列名因 Excel 字段宽度限制被截断，例如：
- `总胆红素-正常范` → 自动修正为 `总胆红素-正常范围`
- `甘油三酯-正常范` → 自动修正为 `甘油三酯-正常范围`

---

### Step 1.5 — 保存 clean 版本

**对象：** 全部 4 个数据源

**操作：**
- 在 Step 1 结果基础上，额外去除所有值全为空的列
- 保存为 `*_clean.csv`，保留所有原始字段（包括姓名、出生日期等标识字段）

**用途：**
提供未加工的参考版本，可用于检查原始数据或与处理结果对比。

---

### Step 2 — 分配患者唯一 ID（person_id）

**匹配键：** 姓名 + 性别

**操作：**
1. 从所有数据源中收集所有（姓名, 性别）组合
2. 为每个唯一组合生成一个 6 位随机字母数字 ID（如 `A3K9PZ`）
3. 生成 `patient_reference.csv`，包含：
   - `person_id`：唯一患者标识
   - `name`、`gender`：匹配字段
   - `dob`：出生日期（仅脑卒中库有）
   - `stroke_diagnosis`：脑卒中诊断（仅脑卒中库有）
   - `in_2022` / `in_2023` / `in_2024` / `in_stroke`：该患者是否出现在对应来源（Y/N）

**注意：**
- 同一患者在同一年度可以有多条记录（多次就诊），不做合并
- 不同人同名同性别会被视为同一人（当前数据无患者编号，无法区分）

---

### Step 2.5 — 检验指标三元组处理（→ 单一数值列）

每个检验指标在原始数据中以三列形式存储：

| 原始列 | 含义 |
|--------|------|
| `血红蛋白` | 检测值 |
| `血红蛋白-异常结果` | 异常标志（↑ 或 ↓） |
| `血红蛋白-正常范围` | 该患者对应的参考范围，如 `110~150` |

**处理方式：** 将三列合并为一列 `血红蛋白_值`，删除原始三列。

#### 数值型指标（28 个）— 区间归一化

**公式：**

```
值 = (检测值 - 下界) / (上界 - 下界)
```

| 结果范围 | 含义 |
|----------|------|
| `0.0 ~ 1.0` | 在正常范围内（0 = 恰好等于下界，1 = 恰好等于上界） |
| `< 0` | 低于正常下界（如 `-0.18` 表示低于下界 18%） |
| `> 1` | 高于正常上界（如 `1.10` 表示超出上界 10%） |
| 空（NaN） | 原始值缺失，或正常范围无法解析 |

**正常范围解析：**

Python 正则解析（不消耗 API）处理所有标准格式：

| 原始字符串 | 解析结果 |
|------------|----------|
| `4.0~10.0` | lower=4.0, upper=10.0 |
| `4.0~10.`（截断小数点）| lower=4.0, upper=10.0 |
| `4.0-10.0~4.0-10.0`（重复）| lower=4.0, upper=10.0 |
| `160~110`（顺序颠倒）| lower=110, upper=160 |
| `>4.0` | lower=4.0（单侧，无法归一化，留空）|
| `阴性`、`>阴性` 等 | 标记为定性（qual=true） |

仅极少数无法识别的格式会调用 Claude API 解析（节省 token）。

#### 定性指标（4 个）— 有序编码

`尿蛋白`、`尿糖`、`尿酮体`、`尿潜血` 转换为 0–5 有序整数：

| 原始值 | 编码 |
|--------|------|
| `-` / `阴性` | 0 |
| `弱阳性` | 1 |
| `阳性(+)` | 2 |
| `阳性(++)` | 3 |
| `阳性(+++)` | 4 |
| `阳性(++++)` | 5 |

---

### Step 2.6 — 多标签列拆分为二值列

#### 人员类型（3 列）

原始列 `人员类型` 的值为逗号分隔的标签组合，如 `"高血压，老年人"`。

拆分为 3 个独立列（有=1，无=0）：

| 新列名 | 含义 |
|--------|------|
| `人员类型_老年人` | 是否为老年人管理对象 |
| `人员类型_高血压` | 是否患有高血压 |
| `人员类型_糖尿病` | 是否患有糖尿病 |

原始 `人员类型` 列删除。

#### 齿列（4 列）

原始列 `齿列` 的值为逗号分隔的牙齿状况组合，如 `"缺齿，龋齿"`。

拆分为 4 个独立列（有=1，无=0）：

| 新列名 | 含义 |
|--------|------|
| `齿列_正常` | 牙齿正常 |
| `齿列_缺齿` | 存在缺齿 |
| `齿列_龋齿` | 存在龋齿 |
| `齿列_义齿(假牙)` | 有义齿 |

原始 `齿列` 列删除。

---

### Step 3 — 去除正常默认值（减小数据量）

以下文本值被替换为空（NaN），因为它们表示"无异常"，对研究无分析价值：

```
无症状  无症状;  正常  无  否  红润  无充血
听见  可顺利完成  未触及  触及正常  齐  无异常
阴性  未见异常  无殊  无特殊  正常范围  未见明显异常
未见  无压痛  软  清  无杂音
```

**注意：** 数值型的检验值（已转换为归一化浮点数）不受此步骤影响。

---

### Step 4 — 去除空列

Step 3 执行后，凡所有行均为空的列一律删除，进一步压缩文件体积。

---

### 最终输出列结构（*_with_id.csv）

| 位置 | 列 | 说明 |
|------|----|------|
| 第 1 列 | `person_id` | 患者唯一 ID，跨文件关联用 |
| 之后 | 原始字段（标识列已删除）| `姓名_x`、`出生日期` 等已移至 patient_reference.csv |
| 检验列 | `血红蛋白_值` 等 | 三元组合并后的归一化值 |
| 人员列 | `人员类型_老年人` 等 | 二值标签列 |
| 牙齿列 | `齿列_缺齿` 等 | 二值标签列 |
